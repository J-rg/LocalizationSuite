BLDatabaseDocument

A basic database storage document that does all the data handling but has no interface support whatsoever.

Conforms to: BLDocumentProtocol
Superclass: NSDocument
Declared In: BLDatabaseDocument.h

Discussion

To add user interaction or other custom functionality, override this class.

addBundle:

Add the bundle to the document. 

- (void)addBundle:(BLBundleObject *)bundle;
Discussion

The bundle will only be added if it is not currently present in the document.

addLanguage:

Add the given language. 

- (void)addLanguage:(NSString *)language;
Discussion

The given language will only be added if it is not currently present in the document. If language is the first to be added, it is automatically assigned to be the reference language.

referenceLanguage

The reference language of the document. 

- (NSString *)referenceLanguage;
Discussion

This language is used by mostly all functions, it is treaded special most of the times. Normally this should be the language that you do your development in. Changes will be transfered FROM this language to all others. So take care when changing it. If no reference language is set, this method returns the defualt reference language, namely @"en".

removeLanguage:

Removes the given language from the document. 

- (void)removeLanguage:(NSString *)language;
Discussion

Please note that this method does not remove the strings for this language, this method removed just the notion that this language exists.

setReferenceLanguage:

Chnage the reference language of the document. 

- (void)setReferenceLanguage:(NSString *)referenceLanguage;
Discussion

Only the language will be changed, no data is affected. However, there is a huge difference in changing the reference language, see -referenceLanguage for details. This method will also add the language to the languages array if it is not yet contained in the document.

updateDocument:

Updates a document file wrapper to the current most version. 

- (BOOL)updateDocument:(NSFileWrapper **)fileWrapper;
Return Value

YES if the document can be opened, NO otherwise.

Discussion

The default method does not present any user prompts, you have to do so in a subclass if you want to. fileWrapper must be non-NULL on return of YES;
