_emitKey:value:comment:

The primitive method to be used by subclasses to emit scanned values. 

- (void)_emitKey:(NSString *)key value:(id)value comment:(NSString *)comment;
Discussion

The order in which this method will be called and in which the keys are being returned is the order in the final file object.

_interpreteFile:

The actual file interpretation happens here. 

- (BOOL)_interpreteFile:(NSString *)path;
Return Value

YES on success, NO on error.

Discussion

This is the primary method to be overridden by subclasses. Whenever a key/value pair is found, _emitKey:value:comment: should be called. The default implementation just throws.

_setForwardsToInterpreter:

Notify the interpreter that is should forward it's emitted keys 

- (void)_setForwardsToInterpreter:(BLFileInterpreter *)aInterpreter;
Discussion

This is to be used out of an interpretation session only. See class description for details about interpreter forwarding. If aInterpreter is nil, it returns to the standard behaviour.
