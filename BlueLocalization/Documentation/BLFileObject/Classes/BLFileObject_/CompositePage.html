BLFileObject

Represents a file in a localization context.

Superclass: BLObject
Declared In: BLFileObject.h

Discussion

Some behaviours depend on the type of the file, especially the extension. This class represents a class cluster. If you want to subclass it, call +registerClass:forPathExtension: registerung your class to be recognized in the process of creating objects.

initWithPath:

Initializes a new file object using the given path. 

- (id)initWithPath:(NSString *)path;
Discussion

The file object's path is the relative portion of the given path. This method also creates a new bundle object with the bundle path portion of the given path. The file object is added to the bundle. Please note that this may result in duplicate bundles that must be resolved afterwards! However, if the path cannot be split into the two portions, the result is a file object with no bundle and it's path set to the given one.

initWithPathExtension:

Initializes a new blank file of the given type. 

- (id)initWithPathExtension:(NSString *)extension;
Discussion

In contrast to initWithPath: this does neither set a path nor create a bundle object.

registerClass:forPathExtension:

Registers a subclass to a file extension. 

+ (void)registerClass:(Class)fileClass forPathExtension:(NSString *)extension;
Discussion

Subclasses should call this one in their +load method, as there are few other possibilites to register a class before it is to be used. Throws, if the class is not a subclass or if the extension has already been registered to any class.
