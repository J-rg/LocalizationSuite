optionsForReferenceFiles

The options that will be set additionally for all interpretations of reference files. 

+ (NSUInteger)optionsForReferenceFiles;
Discussion

By default this method returns BLFileInterpreterAllowChangesToKeyObjects | BLFileInterpreterTrackValueChangesAsUpdate.

optionsForRegularFiles

The options that will be set additionally for all interpretations of non-reference or regular files. 

+ (NSUInteger)optionsForRegularFiles;
Discussion

By default this method returns BLFileInterpreterNoOptions.

stepForInterpertingObjects:withOptions:andLanguages:

Creates a step for interpreting the given set of objects. 

+ (id)stepForInterpertingObjects:(NSArray *)objects 
        withOptions:(NSUInteger)options andLanguages:(NSArray *)languages;
Discussion

The options are additive, which means you can only enable additional options! The options are always modified, i.e. reference language files are imported enabling the modification of keys, whereas other languages are not allowed to.

stepForInterpretingFiles:withOptions:

Creates a step for interpreting the given files. 

+ (id)stepForInterpretingFiles:(NSArray *)files withOptions:(NSUInteger)options;
Discussion

Leaving options empty will use the default settings. The options are always modified, i.e. reference language files are imported enabling the modification of keys, whereas other languages are not allowed to.
