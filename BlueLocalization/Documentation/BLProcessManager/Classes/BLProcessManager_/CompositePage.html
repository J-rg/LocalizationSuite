BLProcessManager

Manages all long duration processings.

Superclass: NSObject
Declared In: BLProcessManager.h

Discussion

Basically this is a wrapper around a processing queue for BLProcessStep objects.

currentStep

Discussion

Can be used to display some interface status information or likewise.

document

Returns the document the process manager belongs to. 

- (NSDocument <BLDocumentProtocol> *)document;
Discussion

Due to the importance of this method to most steps, this one throws if no document has been set beforehand!

enqueueStep:

Enqueues a new step. 

- (void)enqueueStep:(BLProcessStep *)step;
Discussion

This method is thread-safe, as it just forwards to enqueueStepGroup:!

enqueueStepGroup:

Enqueues a new group steps. 

- (void)enqueueStepGroup:(NSArray *)steps;
Discussion

The steps must be independent, concurrent and thread-safe. They might and probably will be executed in parallel. However, groups are performed in the order they were added and are not concurrent. Steps in groups may also add new groups as the they are being performed! This method is thread-safe!

start

Start the processing queue. 

- (void)start;
Discussion

The queue is automatically stopped when all actions have been processed. Resets the number of performed and pending steps.

stop

Stop the processing queue immediatelly. 

- (void)stop;
Discussion

Running operations might be finsihed, however no pending operations will be started. This also clears the queue.
